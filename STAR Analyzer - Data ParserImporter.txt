│ Parser Logic                                                                                                                                                                                                  │
│                                                                                                                                                                                                               │
│ 1. Read file with UTF-8 encoding (fallback to latin-1)                                                                                                                                                        │
│ 2. Parse header lines 1-12 using regex patterns                                                                                                                                                               │
│ 3. Parse scalar variables (A-Z single values)                                                                                                                                                                 │
│ 4. Parse arrays (J, K, L, N, O) in 5-column format                                                                                                                                                            │
│ 5. Trim trailing zeros from arrays (Med-PC padding)                                                                                                                                                           │
│ 6. Validate array lengths match counter values                                                                                                                                                                │
│ 7. Return MedPCSession object with warnings if any issues                                                                                                                                                     │
│                                                                                                                                                                                                               │
│ Error Handling                                                                                                                                                                                                │
│                                                                                                                                                                                                               │
│ - Skip and continue: If a file fails to parse, log the error and continue with remaining files                                                                                                                │
│ - Show summary at end: "Imported X sessions, Y files skipped (errors)"                                                                                                                                        │
│ - Errors logged with filename for troubleshooting                                                                                                                                                             │
│                                                                                                                                                                                                               │
│ Path Memory                                                                                                                                                                                                   │
│                                                                                                                                                                                                               │
│ - Remember last parent folder between sessions (saved to local config)                                                                                                                                        │
│ - Do NOT auto-load - user must click Import each time                                                                                                                                                         │
│ - Pre-populate folder browser with remembered path for convenience                                                                                                                                            │
│                                                                                                                                                                                                               │
│ GUI Features                                                                                                                                                                                                  │
│                                                                                                                                                                                                               │
│ 1. Import Dialog: Browse folder, show progress bar, parse files in background thread                                                                                                                          │
│ 2. Data Viewer: Treeview table showing all sessions (Subject, Date, Active Presses, Licks, etc.)                                                                                                              │
│ 3. Detail View: Double-click session to see full metadata and timestamp arrays                                                                                                                                │

  pass for the day, and if they have met the larger phase criteria, they move onto the next phase. The phases and criteria are listed below:

  1. Magazine Training - 1 day of 100+ licks to proceed to next phase
  2. FR1-30 - 2 consecutive days of 100+ licks to proceed to next phase
  3. FR1-10 - 2 consecutive days of 100+ licks to proceed to next phase
  4. FR5-10 - 2 consecutive days of 100+ licks AND 70% discrimination to proceed to next phase
  5. FR10-10 - once animals are in Test phases, they advance every day, regardless of performance, until they have done a total of 7 days. Then they are DONE.

  If an animal ever fails a phase 3 times in a row, they move back to the previous phase and assume the next day in that cycle. For example, if they passed FR1-30 in 2 days and then eventually return to that
  session after failing the next, they will start with FR1-30 day 3 and need to complete another 2 consecutive days (3+4) to continue again. If an animal ever would move back phases for a third time, they
  are considered a "nonlearner" and removed from the experiment.
